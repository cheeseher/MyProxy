# GitHub 上传速度与 AI 编译器卡顿优化计划

您遇到的问题（TRAE/Cursor 上传 GitHub 慢）是典型的**“终端/Git 流量未走代理”**问题。

通常 Surge Mac 的“增强模式”可以接管大部分流量，但 Git 和终端工具往往需要额外的配置。我们将从三个层面解决这个问题：

## 1. 策略组优化

*   **新增 `🐱 GitHub` 策略组**
    *   **类型**: `select`
    *   **候选项**: `♻️ 自动选择`, `🇭🇰 香港节点`, `🇸🇬 新加坡节点`, `🇺🇸 美国节点`, `🚀 节点选择`。
    *   **逻辑**: GitHub 的服务器主要在美国，但 CDN 遍布全球。通常 `🇭🇰 香港节点` 或 `🇸🇬 新加坡节点` 速度最快。独立出来方便您在上传大文件卡顿时手动切换。

## 2. 规则 (Rule) 覆盖

*   **引入 GitHub 规则集**: 使用 `blackmatrix7/GitHub.list`，涵盖 `github.com`, `githubusercontent.com` 等所有相关域名，强制这些流量走 `🐱 GitHub` 策略组，而不是被误判为直连。

## 3. 终端代理建议 (关键)

除了配置文件，我还会在回复中提供一个**关键的终端命令**。因为即使 Surge 配置好了，有些终端工具（如 Git）默认是不走系统代理的。
我将建议您在 `~/.zshrc` 或 `~/.gitconfig` 中配置代理，这才是彻底解决“上传卡顿”的终极杀招。
