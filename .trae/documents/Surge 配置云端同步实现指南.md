# Surge 配置云端同步实现指南

为了实现您“GitHub 更新配置，手机端自动/手动同步”的需求，我们需要利用 Surge 的 **“托管配置 (Managed Configuration)”** 功能。

核心原理是：不把配置文件当做本地文件，而是当做从云端订阅的一个资源。

## 1. 获取标准订阅链接 (Raw URL)

Surge 无法直接读取 GitHub 的网页，必须使用 **Raw** (原始内容) 链接。
根据您的仓库信息，您的专属订阅链接是：

`https://raw.githubusercontent.com/cheeseher/MyProxy/main/Surge-Dual.conf`

> **注意**：如果您的 GitHub 仓库是 **私有 (Private)** 的，这个链接会失效。您需要生成一个 GitHub Personal Access Token (PAT) 并拼接到链接中，或者简单地将仓库设为 **Public (公开)**（如果配置中不包含敏感密码）。

## 2. 手机端 (iOS) 操作步骤

我将为您提供详细的图文式步骤指南：

1.  **首次安装**: 在 Surge iOS 首页 -> 点击左上角配置名称 -> 选择 **“从 URL 下载配置”** -> 粘贴上面的链接。
2.  **日常更新**: 当您在电脑上修改并推送到 GitHub 后，在手机上再次进入配置列表 -> 点击配置旁的 **“更新”** 按钮（或长按配置选择更新），即可瞬间同步最新规则。

## 3. 进阶：自动更新

虽然 Surge 主配置通常需要手动点击更新，但我会告诉您一个小技巧：利用 `update-url` 参数（我已在配置中预埋，但需要确认生效），让 Surge 知道这个配置文件的“老家”在哪里，方便一键刷新。
