1. 导入配置

您的专属托管配置链接：

https://raw.githubusercontent.com/cheeseher/MyProxy/main/By-Shane.conf

操作步骤

  1. 打开 Surge (iOS/Mac)
  2. 点击配置列表 -> 从 URL 下载配置 / 安装托管配置
  3. 粘贴上方链接并保存
  4. 重要：导入后，请务必修改配置中的 [Proxy Group] -> 我的订阅，填入您的机场订阅地址。

1.1 进阶：自动更新工作流

最佳实践
使用此方法，可以让 Surge 自动更新规则与脚本 (如 TikTok 修复)，同时永远不会覆盖您的私有订阅链接。

操作步骤

  1. 在 Surge 中 创建新配置 -> 创建文本配置。
  2. 清空内容，粘贴下方模板：
#!name=Shane 私有配置 (Local) #!desc=引用云端逻辑，覆盖私有订阅 [General] # 引用云端主配置 (自动保持最新) #!include = https://raw.githubusercontent.com/cheeseher/MyProxy/main/By-Shane.conf [Proxy Group] # 覆盖"我的订阅"策略组，填入你的真实订阅链接 我的订阅 = select, policy-path=您的机场订阅链接, update-interval=0, no-alert=0, hidden=0, include-all-proxies=0

包含 #!include 语句，Surge 会自动合并云端规则与本地订阅。

2. 关键设置：MITM (必读)

为什么需要开启？
如果不开启 MITM，广告拦截、精准分流、脚本功能将全部失效。
  1. 在 Surge 首页点击 "MitM" 模块
  2. 点击 "配置根证书" -> "生成新的 CA 证书"
  3. 点击 "安装证书"，跳转 Safari 下载
  4. 前往 iOS 设置 -> 已下载的描述文件 -> 安装证书
  5. 前往 iOS 设置 -> 通用 -> 关于本机 -> 证书信任设置 -> 开启 Surge 证书信任开关
  1. 点击 Surge 菜单栏图标 -> Dashboard
  2. 左侧栏选择 "MitM"
  3. 点击 "配置根证书" -> "安装证书"
  4. 在“钥匙串访问”中找到 Surge 证书，双击 -> 信任 -> 选择 "始终信任"

6. 常见问题 (Q&A)

Telegram 语音偶尔会断？

请检查 电报消息 策略组是否选了 自动选择。建议手动选择一个固定的 新加坡 节点，并确保节点支持 UDP。

看不到“全局直连”选项？

这是配置的中文别名。如果您只看到 DIRECT，说明配置未更新，请长按配置 -> 立即更新。

GitHub 上传代码慢？

终端需手动设置代理：
export https_proxy=http://127.0.0.1:6152

银行/证券 App 无法登录？

我们已强制 **直连 (Skip Proxy)** 以下应用以修复登录问题：
🏦 银行：中银香港、汇丰、众安、花旗、渣打。
📈 证券：盈立 (uSMART)、富途、复星财富。
无需任何操作，更新配置即刻生效。
